⚠️  CONFIGURAR CREDENCIALES REALES

Opción 1: Usar script rápido
   ./scripts/root/update_odoo_credentials.sh "API_KEY" "HMAC_SECRET"

Opción 2: Editar manualmente
   sudo nano /etc/systemd/system/membresia-relatic.service
   Cambiar ODOO_API_KEY y ODOO_HMAC_SECRET

Luego reiniciar:
   sudo systemctl daemon-reload
   sudo systemctl restart membresia-relatic.service

  VERIFICACIÓN
═══════════════════════════════════════════════════════════════

1. Verificar variables configuradas:
   sudo systemctl show membresia-relatic.service | grep ODOO

2. Probar conexión:
   python3 test_odoo_connection.py

3. Ver logs en tiempo real:
   sudo journalctl -u membresia-relatic.service -f | grep -i odoo

4. Realizar pago de prueba y verificar en Odoo:
   - Contabilidad → Relatic Integration → Logs de Sincronización

═══════════════════════════════════════════════════════════════
  NOTAS IMPORTANTES
═══════════════════════════════════════════════════════════════

- Las credenciales deben coincidir con las configuradas en Odoo:
  * ODOO_API_KEY = relatic_integration.api_key (en Odoo)
  * ODOO_HMAC_SECRET = relatic_integration.hmac_secret (en Odoo)

- El webhook se envía automáticamente cuando se confirma un pago
- Los errores no bloquean el flujo principal
- La integración es idempotente (no crea duplicados)
