Cómo funciona el proceso

En SaaS multi-tenant las plantillas de correo viven primero en la organización Default (id 1). Tu empresa (tenant) no crea filas vacías: clona desde esa base y luego ajusta textos, asuntos y HTML con tu marca.

  1. Comprueba que el panel corresponde a esta organización (menú Empresa, ?organization_id= o subdominio).
  2. Clonar desde base — copia desde Default solo las plantillas que aún no existen aquí.
  3. Editar / Ver — personaliza y previsualiza; el logo del tenant se aplica en la vista previa según configuración.
  4. Clonar y sobrescribir — vuelve a traer asunto y HTML desde Default sobre las plantillas ya existentes (útil si actualizaste la base).

Resetear en una plantilla deja de usar el HTML guardado en BD y vuelve al comportamiento por defecto del código para esa clave.
