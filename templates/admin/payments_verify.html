📋 Información Técnica
Lógica de Match Automático:
  • Canal: Método de pago (Yappy, PayPal, Stripe)
  • Monto: Coincidencia exacta (en centavos)
  • Moneda: Coincidencia exacta
  • Ventana temporal: ±10 minutos (manual) / ±15 minutos (cron)
  • Estado: Transacción debe estar COMPLETED en el proveedor
Protecciones:
  • ✅ Anti-duplicados: Verifica que no exista otro pago con el mismo transaction_id
  • ✅ Match único: Solo hace match automático si hay exactamente 1 candidato
  • ✅ Auditoría: Guarda respuesta completa de la API en yappy_raw_response
  • ✅ Inmutable: payment_reference (YAPPY-XXXXXXXX) NUNCA se sobrescribe
API Endpoint:
POST /api/admin/payments/verify-pending

Requiere autenticación de administrador. Retorna estadísticas detalladas de la verificación.
