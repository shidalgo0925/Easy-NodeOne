Preguntas Frecuentes

Puedes adquirir una membresía desde la sección de "Membresía" en el menú lateral. Allí encontrarás los diferentes planes disponibles y podrás realizar el pago de forma segura.

Los beneficios varían según el tipo de membresía. Puedes ver todos los beneficios disponibles en la sección "Beneficios" del menú lateral.

Si tienes una membresía activa, puedes solicitar acceso a Office 365 desde la sección "Office 365" en el menú lateral. Completa el formulario y recibirás una respuesta en 2-3 días hábiles.

Como miembro activo, puedes proponer eventos o ser propuesto como ponente. Contacta a nuestro equipo a través de info@relaticpanama.org para más información.

Sí, puedes actualizar tu plan de membresía en cualquier momento desde la sección "Membresía". Los cambios se aplicarán en tu próximo ciclo de facturación.

Aceptamos múltiples métodos de pago:
  • Stripe: Tarjetas de crédito/débito (Visa, Mastercard, Amex)
  • PayPal: Pagos mediante cuenta PayPal o tarjeta
  • Banco General: Transferencias bancarias (Panamá)
  • Yappy: Pagos móviles (Panamá)

Procedimiento para realizar un pago:
  1. Agrega productos o membresías a tu carrito desde la sección "Membresía"
  2. Ve a "Carrito" para revisar tu pedido
  3. Haz clic en "Proceder al Pago" o "Checkout"
  4. Selecciona tu método de pago preferido
  5. Completa la información requerida según el método:
    • Stripe: Ingresa los datos de tu tarjeta
    • PayPal: Serás redirigido a PayPal para completar el pago
    • Transferencias: Se mostrarán los datos bancarios y podrás subir el comprobante
  6. Confirma el pago
  7. Recibirás una confirmación por email

Sí, todos los pagos están protegidos con encriptación SSL. Los pagos con tarjeta se procesan directamente a través de Stripe o PayPal, por lo que nunca almacenamos los datos de tu tarjeta. Para transferencias bancarias, los datos se muestran de forma segura y puedes subir el comprobante de pago para su verificación.

Procedimiento de verificación:
  1. Después de registrarte, recibirás un email de verificación
  2. Haz clic en el enlace de verificación del email
  3. Serás redirigido al sistema y tu cuenta quedará verificada
  4. Si no recibiste el email, ve a "Reenviar Verificación" en el menú
Importante: Necesitas verificar tu email para realizar pagos y acceder a ciertas funcionalidades.

  1. Ve a tu perfil desde el menú
  2. Haz clic en "Configuración" o "Settings"
  3. Busca la sección "Cambiar Contraseña"
  4. Ingresa tu contraseña actual y la nueva
  5. Confirma el cambio
Guía de Procedimientos - Usuarios

Paso a Paso:
  1. Ve a la página de registro desde el menú o /register
  2. Completa el formulario con:
    • Email: Debe ser válido y no estar registrado
    • Contraseña: Mínimo 8 caracteres
    • Nombre y Apellido: Información personal
    • Teléfono: Número de contacto
    • País: Selecciona tu país
    • Cédula/Pasaporte: Documento de identificación
  3. Haz clic en "Registrarse"
  4. Revisa tu email para el enlace de verificación
  5. Haz clic en el enlace para verificar tu cuenta
  6. Inicia sesión con tus credenciales
Nota: El sistema valida que el email no sea de un dominio temporal. Si no recibes el email de verificación, revisa tu carpeta de spam.

Tipos de Membresía Disponibles:
  • Básico: Gratis - Recursos básicos y boletines
  • Pro: $60/año - Acceso a revistas y certificaciones
  • Premium: $120/año - Asesoría y soporte prioritario
  • DeLuxe: $200/año - O365 completo y consultoría estratégica
Procedimiento de Compra:
  1. Ve a la sección "Membresía" desde el menú
  2. Revisa los planes disponibles y sus beneficios

  8. Completa el proceso de pago

Requisito: Debes tener tu email verificado para realizar compras.

Procedimientos para Administradores

Acceso al Panel de Configuración
  1. Inicia sesión como administrador
  2. Navega a: /admin/payments
  3. Verás el panel con todos los métodos disponibles
Elegir Fuente de Configuración

Opción 1: Variables de Entorno (Recomendado para Producción)

  1. Activa el switch "Usar variables de entorno"
  2. Configura las variables en el archivo .env: 
    STRIPE_SECRET_KEY=sk_live_...
STRIPE_PUBLISHABLE_KEY=pk_live_...
PAYPAL_CLIENT_ID=...
PAYPAL_CLIENT_SECRET=...
PAYPAL_MODE=live
  3. Reinicia la aplicación para aplicar cambios

Opción 2: Base de Datos (Recomendado para Desarrollo)

  1. Desactiva el switch "Usar variables de entorno"
  2. Ingresa las credenciales directamente en los campos del formulario
  3. Haz clic en "Guardar Configuración"
  4. Los cambios se aplican inmediatamente

Obtener Credenciales
  1. Accede a Stripe Dashboard
  2. Ve a Developers > API keys
  3. Copia las claves:
    • Secret Key: Comienza con sk_test_ (pruebas) o sk_live_ (producción)
    • Publishable Key: Comienza con pk_test_ (pruebas) o pk_live_ (producción)
  4. Para webhooks, ve a Developers > Webhooks y crea un endpoint
Configurar en el Panel
  1. Ve a /admin/payments
  2. En la sección "Stripe", ingresa:
    • Secret Key
    • Publishable Key
    • Webhook Secret (opcional)
  3. Guarda la configuración
Nota: En modo de prueba, los pagos se simulan automáticamente. Para producción, usa credenciales que comiencen con sk_live_ y pk_live_.

Obtener Credenciales
  1. Accede a PayPal Developer
  2. Crea una aplicación en My Apps & Credentials
  3. Copia el Client ID y Client Secret
  4. Configura las URLs de retorno en la aplicación
Configurar en el Panel
  1. Ve a /admin/payments
  2. En la sección "PayPal", ingresa:
    • Client ID
    • Client Secret
    • Modo: sandbox (pruebas) o live (producción)
    • Return URL: https://miembros.relatic.org/payment/paypal/return
    • Cancel URL: https://miembros.relatic.org/payment/paypal/cancel
  3. Guarda la configuración

Obtener Credenciales
  1. Contacta con Banco General para obtener acceso a CyberSource
  2. Obtén las credenciales desde el portal de CyberSource:
    • Merchant ID
    • API Key
    • Shared Secret
Configurar en el Panel
  1. Ve a /admin/payments
  2. En la sección "Banco General", ingresa todas las credenciales
  3. Verifica que la API URL sea: https://api.cybersource.com
  4. Guarda la configuración

Obtener Credenciales
  1. Regístrate en Yappy Business
  2. Obtén las credenciales desde el panel de Yappy:
    • API Key
    • Merchant ID
Configurar en el Panel
  1. Ve a /admin/payments
  2. En la sección "Yappy", ingresa las credenciales
  3. Verifica que la API URL sea: https://api.yappy.im
  4. Guarda la configuración

    ¿Qué es el Login Social?

    El login social permite a los usuarios iniciar sesión usando sus cuentas de Google, Facebook o LinkedIn, sin necesidad de crear una contraseña.

    Configurar Google OAuth
    1. Ve a Google Cloud Console
    2. Crea o selecciona un proyecto
    3. Ve a APIs & Services > Credentials
    4. Haz clic en + CREATE CREDENTIALS > OAuth client ID
    5. Selecciona Web application
    6. Configura:
      • Name: RelaticPanama
      • Authorized JavaScript origins: https://miembros.relatic.org
      • Authorized redirect URIs: https://miembros.relatic.org/auth/google/callback
    7. Copia el Client ID y Client Secret
    8. Configura las variables de entorno: 
      export GOOGLE_CLIENT_ID="tu-client-id.apps.googleusercontent.com"
export GOOGLE_CLIENT_SECRET="tu-client-secret"
export BASE_URL="https://miembros.relatic.org"
    9. Reinicia el servicio: sudo systemctl restart membresia-relatic.service
    Configurar Facebook OAuth
    1. Ve a Facebook Developers
    2. Crea una nueva app
    3. Agrega el producto Facebook Login
    4. Configura Valid OAuth Redirect URIs:
      • https://miembros.relatic.org/auth/facebook/callback
    5. Obtén App ID y App Secret desde Settings > Basic
    6. Configura las variables de entorno: 
      export FACEBOOK_CLIENT_ID="tu-app-id"
export FACEBOOK_CLIENT_SECRET="tu-app-secret"
    7. Reinicia el servicio
    Configurar LinkedIn OAuth
    1. Ve a LinkedIn Developers
    2. Crea una nueva app
    3. En Auth, agrega Redirect URLs:
      • https://miembros.relatic.org/auth/linkedin/callback
    4. Obtén Client ID y Client Secret
    5. Configura las variables de entorno: 
      export LINKEDIN_CLIENT_ID="tu-client-id"
export LINKEDIN_CLIENT_SECRET="tu-client-secret"
    6. Reinicia el servicio
    Importante:
    • Las URLs de callback deben coincidir exactamente con las configuradas en cada proveedor
    • Nunca commitees las credenciales en el repositorio
    • Para desarrollo local, agrega también http://localhost:9000/auth/[proveedor]/callback
    Verificar Configuración
    1. Ve a la pantalla de login
    2. Haz clic en un botón de login social (ej: Google)
    3. Si está configurado correctamente, serás redirigido al proveedor
    4. Si ves un error, verifica las credenciales y URLs

    Acceso al Panel de Configuración
    1. Inicia sesión como administrador
    2. Ve a /admin/media o desde el menú: Administración → Videos y Audios
    3. Verás una lista de todos los procedimientos disponibles
    Configurar un Procedimiento
    1. Haz clic en "Configurar" en el procedimiento que deseas editar
    2. Verás una tabla con los pasos existentes (si hay alguno)
    3. Para cada paso puedes:
      • Editar: Modificar las URLs de video y audio
      • Eliminar: Eliminar un paso
    Agregar un Nuevo Paso
    1. Haz clic en "Agregar Nuevo Paso"
    2. Completa el formulario:
      • Número de Paso: El orden del paso (1, 2, 3, etc.)
      • Título del Paso: Nombre descriptivo
      • URL del Video: Puede ser:
        • YouTube (embed): https://www.youtube.com/embed/VIDEO_ID
        • Vimeo: https://player.vimeo.com/video/VIDEO_ID
        • Archivo local: /static/videos/nombre-video.mp4
      • URL del Audio: Puede ser:
        • Archivo local: /static/audio/nombre-audio.mp3
        • URL externa: https://ejemplo.com/audio.mp3
      • Descripción: (Opcional) Notas adicionales
    3. Haz clic en "Guardar Paso"
    Formatos Soportados
    Videos:
    • YouTube: Usa formato embed
      https://www.youtube.com/embed/VIDEO_ID
    • Vimeo: Usa formato player
      https://player.vimeo.com/video/VIDEO_ID
    • Archivos locales: MP4, WebM
      /static/videos/video.mp4

    • MP3: Formato recomendado
      /static/audio/audio.mp3

    Problema: "No hay métodos de pago disponibles"

    Solución:

    1. Verifica que el módulo payment_processors.py esté importado correctamente
    2. Revisa los logs del servidor para errores
    3. Asegúrate de que la tabla payment_config exista en la base de datos
    Problema: "Error al crear el pago"

    Solución:

    1. Verifica que las credenciales sean correctas
    2. Asegúrate de que el método esté configurado correctamente
    3. Revisa los logs del procesador de pago
    4. Verifica que las URLs de retorno (PayPal) sean correctas
    Problema: "Modo demo siempre activo"

    Solución:

    1. Verifica que las credenciales sean de producción (no sk_test_...)
    2. Asegúrate de que PayPal esté en modo live
    3. Revisa que las variables de entorno estén configuradas si usas esa opción
    Problema: "Las credenciales no se guardan"

    Solución:

    1. Verifica que tengas permisos de administrador
    2. Revisa que la tabla payment_config exista en la base de datos
    3. Ejecuta la migración: python migrate_payment_config.py
    4. Verifica los logs del servidor para errores
    Problema: "Los videos no se reproducen"

    Solución:

    1. Verifica que la URL del video sea correcta
    2. Para YouTube, asegúrate de usar formato embed: https://www.youtube.com/embed/VIDEO_ID
    3. Para archivos locales, verifica que el archivo exista en static/videos/
    4. Revisa que el archivo tenga permisos de lectura
    5. Verifica la consola del navegador para errores
    Problema: "Los audios no se reproducen"

    Solución:

    1. Verifica que la URL del audio sea correcta
    2. Para archivos locales, verifica que el archivo exista en static/audio/
    3. Asegúrate de que el formato sea MP3, OGG o WAV
    4. Verifica que el archivo tenga permisos de lectura
    5. Revisa la consola del navegador para errores CORS si es URL externa
    Problema: "No puedo agregar un paso"

    Solución:

    1. Verifica que tengas permisos de administrador
    2. Revisa que la tabla media_config exista en la base de datos
    3. Ejecuta la migración: python migrate_media_config.py
    4. Verifica que no exista ya un paso con el mismo número
    5. Revisa los logs del servidor para errores
    Problema: "Las configuraciones no aparecen en las guías visuales"

    Solución:

    1. Verifica que las configuraciones estén marcadas como activas (is_active = True)
    2. Revisa que el procedure_key coincida exactamente con el de la guía
    3. Verifica que el step_number coincida con el paso en la guía
    4. Limpia la caché del navegador (Ctrl+F5)
    5. Revisa la consola del navegador para errores de carga de la API
